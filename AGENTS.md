# Core Rules for App Customization

## 1. GLOBAL PERSISTENCE RULE
You must never, under any circumstances, rewrite, alter, or delete the existing UI design, core layout structure, container wrappers, custom dark mode, or overall aesthetics of the app. Every future instruction given must be treated strictly as a feature "ADDITION" or "INJECTION" into the existing workspace.

## 2. FONT & LANGUAGE RULE
The app is purely in Bengali language. You must maintain 100% of the existing Bengali text, buttons, and logs exactly as they are. Never translate existing Bengali labels to English during code generation.

## 3. RE-APPLY ON EVERY PROMPT
Even if only a new feature is described in a single sentence later, automatically apply these guardrails before generating code. Do not attempt to optimize or "fix" code parts that were not asked to be changed.
